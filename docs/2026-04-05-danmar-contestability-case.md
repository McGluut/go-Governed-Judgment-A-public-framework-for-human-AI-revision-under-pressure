# 2026-04-05 Danmar Contestability Case

## Specification

### Problem Definition
Run the first real-case test of `docs/contestability-and-exit-layers-note.md` against an actual outside-pressure event:

Danmar's public challenge:
- where are the breathable exits?
- how is governance distinguished from management when one side still holds more of the mechanisms?

The question here is not merely whether the wording was answered.

It is:
- does the relation, as actually exposed publicly, count as `challengeable governance`?
- or only as `managed participation` under mechanism asymmetry?

### Constraints
- Treat the public relation itself as the object under test.
- Do not overread public visibility as real contestability.
- Do not ignore real delta just because full symmetry is absent.
- Do not use private Agora/product details.

### Unknowns
- How much public re-entry is available in practice across X and GitHub surfaces.
- Whether public git history plus public criticism count as sufficient continuity preservation.
- Whether one real case is enough to classify the public relation firmly rather than provisionally.

### Invariants
- Repo owner still controls the constitutional text, repo logging, and promotion surface.
- The outside critic still lacks equal control over storage, admission, and follow-up action.
- Public visibility lowers silent capture risk, but does not eliminate asymmetry.

## Plan

### Strategy
Run a bounded Agora pass through four checks:
- mechanism map
- threshold test
- delta test
- classification

## Mechanism Map

### Owner-controlled machinery

- repo text and promotion surface
- durable storage inside the repo
- selection of which external pressure becomes first-class repo artifact
- branch, merge, and publication control
- framing of the follow-up note and its placement in the truth architecture

### Outside-pressure surface available to Danmar

- public challenge text
- public continuation through X
- indirect pressure through public visibility and reputational exposure
- possible external fork or commentary outside the repo

### Machinery Danmar does not control

- repo storage admission
- SEEKING_LOG inclusion
- constitutional or docs-layer promotion
- review path inside the repo
- deployment or interface governance beyond public visibility

## Agora Pass

### Guardian

Main judgment:
- this case is stronger than a rhetorical reply cycle;
- but it does not yet count as fully challengeable governance.

Why:
- Danmar could challenge the framework publicly;
- the challenge did alter the repo's public truth surface;
- but the machinery of storage, admission, and follow-up remained owner-controlled throughout.

Strongest warning:
- public responsiveness can still function as elegant containment if every serious challenge must pass back through the same owner-curated machinery.

### Seeker

Main judgment:
- the case did produce real delta.

Observed delta:
- governance-capture challenge note
- multiple Agora passes
- refined pressure classification
- operational distinction set
- new `contestability and exit layers` note

This means the challenge was not merely acknowledged.
It changed the repo.

But:
- the changed artifacts remain generated, stored, and interpreted by the same dominant pole.
- the case therefore pressures machinery asymmetry more than it resolves it.

### Builder

Main judgment:
- the right classification is `mixed / provisional`, not clean success.

Threshold read against `docs/contestability-and-exit-layers-note.md`:

#### Positive conditions

1. `Machinery can be challenged`
   - yes, at least at the text/governance-description layer
2. `Contest changes the path`
   - yes, clearly
3. `Loss is disclosed`
   - partially only; still underworked
4. `Exit is not disappearance by default`
   - not yet established
5. `Re-entry is non-opaque`
   - only partially, because public continuation exists but repo-internal promotion remains owner-controlled
6. `History is not silently monopolized`
   - partially yes, because public history and append-only logging reduce silent capture

Result:
- enough to reject pure dismissal
- not enough to count as full challengeable governance

#### Hard-failure conditions

None are fully proven in this case.

But one near-failure remains live:
- `content-only challenge` is no longer fully true
  because machinery was explicitly challenged and that challenge changed the public truth surface
- yet the machinery challenge still lacked its own independent contest path

#### Soft-degradation signals

- challenge is still routed back through owner-controlled storage and interpretation
- loss allocation and portability remain opaque
- contest of machinery is named, but not yet independently operable

These are sufficient to block a clean `challengeable governance` verdict for now.

## Classification

Best current judgment:

`Mixed / provisional`

More explicit form:

`challengeable governance at the public text-pressure layer, but not yet at the machinery layer`

Negative form:
- not mere managed participation in the shallow sense, because the challenge produced real inspectable delta
- not yet robust challengeable governance, because machinery asymmetry remains unworked in practice

## Current Verdict

Danmar's case shows:
- the framework can take real outside pressure without merely defending itself;
- public criticism can alter the repo's truth surface;
- but the current public relation still stops short of full contestable governance because storage, admission, and follow-up remain structurally asymmetric.
