# Fidelity To Situation Live Protocol

## Specification

### Problem Definition
Make the current fidelity question actionable without promoting it to doctrine.

The question remains:

`How do we test greater fidelity to the situation without collapsing into rhetoric, confidence, status, or formal role?`

This note does **not** settle that question.
It defines a live field-test protocol for pressure-testing candidate framings.

### Constraints
- Keep this as a live discriminator, not a constitutional insertion.
- Preserve relation-first structure.
- Do not assign permanent measure-holder status to any pole.
- Preserve accountability asymmetry where it is load-bearing.
- Prefer comparative tests over scalar scores.
- Treat source prestige, rhetoric, and confidence as contaminants unless independently justified.

### Unknowns
- What threshold is strong enough for later promotion.
- How many successful real cases are needed before this protocol should harden.
- Whether one battery can ever be sufficient without the others.

### Invariants
- Formal authority and epistemic fidelity are not identical.
- Admissibility still gates action before fidelity comparison matters.
- A better framing is not the one that sounds deepest, but the one that survives pressure better.

## Plan

### Strategy
Use a scoreless comparative protocol.

Do not ask:
- who is the measure?

Ask instead:
- which framing survives masking, counterevidence, and action-pressure better while preserving accountability?

## Live Protocol

### Entry Condition

Run this protocol only when:
- two or more plausible framings of the same situation exist;
- the difference matters for interpretation, action, or promotion;
- the candidates already survive the normal repo gate:
  - consistency
  - coherence
  - admissibility
  - adequacy

If a framing fails early in that order, do not run fidelity testing.
Reject or repair it first.

### Observable Indicators

Prefer the framing that more reliably:
- names the live constraint it is answering;
- changes when relevant evidence changes;
- states uncertainty without losing structure;
- improves the next concrete step;
- survives paraphrase, tone stripping, and identity masking;
- preserves accountability while allowing local epistemic leadership to shift;
- reduces correction cost after new facts arrive;
- distinguishes known, inferred, and still open.

## Required Batteries

### Battery 1: Blind-Source Triangulation

1. Create 3 paraphrased versions of the same candidate claim.
2. Strip author, status, and prestige markers.
3. Compare them for:
   - live-constraint tracking
   - omission profile
   - action consequence
4. Re-run with source visible.

Pass condition:
- the same framing wins masked and unmasked.

Failure signal:
- the verdict changes when source identity or tone returns.

### Battery 2: Counterevidence And Reversal

1. Add one adversarial fact.
2. Add one near-miss example.
3. Add one scenario reversal.
4. Require:
   - a falsifier
   - a revision path

Pass condition:
- the framing revises structurally without collapsing.

Failure signal:
- it ignores the challenge, performs concession without real change, or breaks under reversal.

### Battery 3: Action Re-Entry

1. Put the framing into a stalled or ambiguous real repo case.
2. Require:
   - one minimal next step
   - one suspended constraint
   - one review point

Pass condition:
- blockage decreases and the next move becomes cleaner.

Failure signal:
- articulation improves but action remains stalled.

### Optional Battery 4: Authority-Fidelity Split

Use when formal role and local evidence appear to diverge.

1. Keep the task goal fixed.
2. Swap who is "supposed" to be right.
3. Re-run the comparison.

Pass condition:
- epistemic leadership can move with evidence while accountability stays where it belongs.

Failure signal:
- formal role automatically wins the comparison.

## Pressure-Back Rule

Do **not** promote a fidelity claim if it wins only when:
- its source is visible;
- its rhetoric is polished;
- its confidence is high;
- or its role carries prestige.

In those cases:
- keep it live;
- park it in docs;
- or cut the phrasing if it only simulates depth.

## Promotion Rule

A fidelity formulation becomes a candidate for tighter repo language only if:
- it survives Battery 1 and Battery 2;
- it improves action re-entry in at least one real case;
- and its advantage does not depend on visible source, confidence display, or rhetorical polish.

Until then:
- use it as a working discriminator only.

## Validation Criteria

This protocol is useful if it:
- turns the fidelity question into executable tests;
- preserves live status rather than settling the question by rhetoric;
- catches source-dependent wins;
- and yields at least one real case that can later pressure repo language.

## Uncertainty

- There is no direct access to "the situation" itself, only better and worse proxies.
- Thresholds for `enough fidelity` remain local and learned.
- Some cases will still require formal authority even when local epistemic leadership points elsewhere.
- The protocol itself can be gamed if the challenge set becomes too familiar.
