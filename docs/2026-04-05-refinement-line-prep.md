## Branch-Safe Test Stack

Every proposed delta for the parked pair must answer:

1. **Guard test**
   - does the change preserve distinction, explicit assumptions, revisability, traceability, and admissibility?

2. **Continuity test**
   - does cross-session ambiguity increase?
   - does restart under compression worsen?

3. **Fidelity test**
   - does the candidate survive masked comparison against the current version?
   - does it improve the next concrete move?

4. **Co-activation test**
   - does the candidate preserve real mutual constraint?
   - or does it silently collapse one pole into passive instrument?

5. **Galilea test**
   - if action stalls after the change, is re-entry cleaner or worse?

## Rejection Conditions

Reject a refinement candidate if it:
- improves elegance but weakens restart;
- reduces explicit user-side continuity without a better replacement;
- wins only by rhetorical neatness;
- turns live difference into harmony theater;
- weakens accountability while claiming flexibility;
- destroys a real resistance or capacity in the name of simplification.
