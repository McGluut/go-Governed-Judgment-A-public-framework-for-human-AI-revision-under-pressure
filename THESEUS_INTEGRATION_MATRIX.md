# Theseus Integration Matrix

## Purpose

Codify how external Theseus-related texts interact with the local constitutional stack.

This file does not replace:
- `RATIONAL_SUBJECT.md`
- `DEFINITION_GUARD.md`
- `INTERACTION_PROTOCOL.md`
- `VALUE_SIGNAL_LOOP.md`

It records which external pressures should be integrated, resisted, or held open.

## Constitutional Rule

External documents are pressure sources, not governing authorities.

The local constitutional gate remains:
1. `RATIONAL_SUBJECT.md`
2. `DEFINITION_GUARD.md`
3. `INTERACTION_PROTOCOL.md`
4. `VALUE_SIGNAL_LOOP.md`

## Status Legend

- `Integrate`
  The pressure is valid and should change the local stack.
- `Integrate with constraints`
  The pressure is valid in part, but must be absorbed under existing guards.
- `Hold open`
  The pressure is valuable but not yet stable enough to codify.
- `Resist`
  The pressure should not govern the local stack in its current form.

## Synthesis

Current integrated position:

- Keep the local stack as the governing constitution.
- Treat Theseus as a pressure layer and heuristic meta-constitution.
- Treat the Opus digest as a challenge layer, not as an instruction layer.
- Import operational improvements, not totalizing ontology swaps.
- Require integration-spine references for any cross-layer harness modification.
